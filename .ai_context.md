# AI Context — Friends & Family Billing

## Project Name
Friends & Family Billing (Family Bill Splitter)

## Project Type
Cloud-based web application — Firebase Hosting + Firestore + Cloud Functions + Firebase Auth

## Primary Languages
- JavaScript (Vanilla ES6+, esbuild-bundled)
- HTML5
- CSS3

## Key Entry Points

| File | Purpose |
|------|---------|
| `src/main.js` | All application logic (~4,690 lines) — the primary source file |
| `src/index.js` | Bundle entry point — imports from main.js, assigns functions to window.* for inline handler compatibility |
| `src/platform/firebase.js` | Bridge module — re-exports Firebase globals from window.* into ES module imports |
| `functions/index.js` | Cloud Functions v2 entry point — resolveShareToken, submitDispute, getEvidenceUrl, submitDisputeDecision |
| `firebase-config.js` | Initializes Firebase from local config, assigns auth/db/storage/analytics to window.* |
| `AGENTS.md` | AI agent instructions and behavioral rules |

**Note:** `script.js` is a build artifact (gitignored). Source of truth is `src/`. Always edit `src/main.js`, never `script.js`.

| Directory | Justification |

| `dist/` | The build artifact is `script.js` at the repo root (gitignored). No separate dist/ directory. The esbuild output goes directly to root per Firebase Hosting config (public directory = `.`). |

## Do Not Touch

| File/Path | Reason |
|-----------|--------|
| `script.js` | Build artifact — never edit directly; run `npm run build` to regenerate |
| `script.js.map` | Source map — auto-generated with build |
| `firebase-config.js` | Must keep placeholder values only; real config is in gitignored `firebase-config.local.js` |
| `firestore.rules` | Security rules — changes affect data access for all users; require careful review |
| `storage.rules` | Storage security rules — same caution as above |
| `stamp-version.js` | Predeploy script — runs automatically as part of deploy |

## High-Risk Areas (Require Human Review Before Any Agent-Proposed Change)

| Area | Risk |
|------|------|
| `calculateAnnualSummary()` | Bill splitting math — errors affect real financial amounts for real users |
| `recordPayment()` / `deletePaymentEntry()` | Ledger integrity — payments are append-only; errors are hard to undo |
| `getPaymentTotalForMember()` | Balance derivation — wrong totals mislead users about what they owe |
| `firestore.rules` | Data access security — incorrect rules can expose or corrupt user data |
| Cloud Functions auth/token validation | Public-facing endpoint security |

## Standard Compliance Notes

- **`dist/` absent:** Intentional. Build artifact (`script.js`) lives at repo root per Firebase Hosting config. Documented above.
- **`scripts/ci/` present but not connected to CI:** CI enforcement scripts exist per the standard. No CI pipeline is connected. Run scripts locally before commits. `scripts/check-no-public-secrets.mjs` is also run as part of `npm test`.
- **`.cursor/rules/deploy.mdc`:** Valid Cursor configuration file. Mirrors `DEPLOYMENT.md` deploy conventions.
- **`firebase-config.local.js`:** Gitignored. Contains real Firebase web config. Never commit this file.
- **`script.js` and `script.js.map`:** Gitignored. Build artifacts. Never commit these files.
