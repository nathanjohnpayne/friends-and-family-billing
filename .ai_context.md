# AI Context — Friends & Family Billing

## Project Name
Friends & Family Billing (Family Bill Splitter)

## Project Type
Cloud-based web application — React SPA on Firebase Hosting + Firestore + Cloud Functions + Firebase Auth

## Primary Languages
- JavaScript (React + Vite)
- CSS3 (design tokens)

## Key Entry Points

| File | Purpose |
|------|---------|
| `src/app/main.jsx` | React app entry point |
| `src/app/App.jsx` | Root component — routing, auth, lazy-loaded views |
| `src/lib/BillingYearService.js` | Service-owns-state pattern — all billing mutations |
| `src/lib/firebase.js` | Modular Firebase initialization (reads from .env.local) |
| `functions/index.js` | Cloud Functions v2 — resolveShareToken, submitDispute, getEvidenceUrl, submitDisputeDecision |
| `AGENTS.md` | AI agent instructions and behavioral rules |

## Architecture

- **Service-owns-state**: `BillingYearService` owns canonical mutable state; React subscribes via `useSyncExternalStore`
- **Mutations**: All data changes go through service methods → `_setState()` → subscriber notification → React re-render
- **Code-split**: Heavy views and dialogs loaded via `React.lazy()` + `<Suspense>`
- **Config**: Firebase config in `.env.local` (gitignored) as `VITE_FIREBASE_*` vars

## Do Not Touch

| File/Path | Reason |
|-----------|--------|
| `firestore.rules` | Security rules — changes affect data access for all users |
| `storage.rules` | Storage security rules — same caution as above |
| `.env.local` | Real Firebase config — gitignored, never commit |

## High-Risk Areas (Require Human Review)

| Area | Risk |
|------|------|
| `src/lib/calculations.js` — `calculateAnnualSummary()` | Bill splitting math — errors affect real financial amounts |
| `src/lib/BillingYearService.js` — `recordPayment()` / `reversePayment()` | Ledger integrity — payments are append-only |
| `src/lib/calculations.js` — `getPaymentTotalForMember()` | Balance derivation — wrong totals mislead users |
| `firestore.rules` | Data access security |
| Cloud Functions auth/token validation | Public-facing endpoint security |
