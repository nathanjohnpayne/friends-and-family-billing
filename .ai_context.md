## Architecture

- **React-only**: React SPA at `/`, backed by Firestore data and Firebase Auth.
- **Config bridge**: `firebase-config.local.js` sets `window.__FIREBASE_CONFIG__` — used by the React app. Holds Firebase config (no longer carries `logodevKey`). Gitignored, must exist locally and be deployed.
- **Service-owns-state**: `BillingYearService` owns canonical mutable state; React subscribes via `useSyncExternalStore`
- **Code-split**: Heavy views and dialogs loaded via `React.lazy()` + `<Suspense>`
- **Two-step build**: `npm run build` runs `build:react` (Vite builds the root `index.html` React entry → `app/`) then `build:assemble` (`node scripts/assemble-deploy.js`: generates the runtime `firebase-config.local.js` bridge and copies shared assets into `app/`)
- **Tests**: `npm test` runs `test:react` (Vitest — 1029 tests across 46 files) then `test:functions` (58 Cloud Functions tests via Node's native test runner) then `test:secrets`. Playwright E2E tests available via `npm run test:e2e`.

| File/Path | Reason |
|-----------|--------|
| `firestore.rules` | Security rules — changes affect data access for all users |

## High-Risk Areas (Require Human Review)

| Area | Risk |
|------|------|
| `src/lib/calculations.js` — `calculateAnnualSummary()` | Bill splitting math — errors affect real financial amounts |
| `src/lib/BillingYearService.js` — `recordPayment()` / `reversePayment()` | Ledger integrity — payments are append-only |
| `src/lib/calculations.js` — `getPaymentTotalForMember()` | Balance derivation — wrong totals mislead users |
| `firestore.rules` | Data access security |
| Cloud Functions auth/token validation | Public-facing endpoint security |
